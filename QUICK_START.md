# Быстрый старт: от GitHub до RuStore за 30 минут

## Что у тебя есть прямо сейчас

✅ Подключён GitHub (репозиторий с кодом Nova)
✅ Готов workflow для сборки AAB (`github-workflows/android-build.yml`)
✅ Все исправления безопасности под RuStore сделаны
✅ Тестовый аккаунт для модератора работает

## Что нажать прямо сейчас — 7 шагов

### Шаг 1. Найди свой репозиторий (1 минута)

1. Открой https://github.com
2. Аватар справа вверху → **Your repositories**
3. Найди репозиторий с кодом Nova → открой
4. Запомни URL — он будет нужен для всех следующих шагов:
   `https://github.com/ТВОЙ_USERNAME/НАЗВАНИЕ_РЕПО`

---

### Шаг 2. Создай workflow для сборки (2 минуты)

1. В репозитории нажми кнопку **Add file** (вверху справа) → **Create new file**
2. В поле имени введи **точно так**:
   ```
   .github/workflows/android-build.yml
   ```
3. В содержимое **скопируй весь** код из файла
   `github-workflows/android-build.yml` (открой его в poehali)
4. Внизу страницы → **Commit changes**

---

### Шаг 3. Создай ключ подписи (5 минут, один раз)

Установи JDK 17 на свой компьютер (https://adoptium.net/) — это бесплатно.

Открой терминал (Windows: PowerShell, Mac: Terminal) и выполни:

```bash
keytool -genkey -v -keystore nova-release.keystore -alias nova \
  -keyalg RSA -keysize 2048 -validity 10000
```

Ответь на вопросы:
- Имя/фамилия → как угодно (например: `Nova Developer`)
- Организация → название компании или ник
- Город, страна → по факту
- **Придумай 2 пароля** (запомни их обязательно!):
  - Пароль keystore
  - Пароль key (можно сделать таким же)

⚠️ **Файл `nova-release.keystore` и оба пароля сохрани в надёжное место.** Без них нельзя обновлять приложение в магазинах.

---

### Шаг 4. Закодируй ключ в base64 (1 минута)

**Mac / Linux:**
```bash
base64 nova-release.keystore | tr -d '\n' > keystore-base64.txt
```

**Windows (PowerShell):**
```powershell
[Convert]::ToBase64String([IO.File]::ReadAllBytes("nova-release.keystore")) | Out-File keystore-base64.txt
```

Откройте файл `keystore-base64.txt` — там длинная строка букв и цифр. Это значение для одного из секретов.

---

### Шаг 5. Добавь 4 секрета в GitHub (3 минуты)

В своём репозитории GitHub:
1. Вкладка **Settings** (вверху, не в профиле, а в репозитории)
2. Слева в меню: **Secrets and variables** → **Actions**
3. Кнопка **New repository secret**
4. Создай 4 секрета по очереди:

| Имя секрета | Что вставить |
|-------------|--------------|
| `ANDROID_KEYSTORE_BASE64` | Содержимое файла `keystore-base64.txt` (длинная строка) |
| `ANDROID_KEYSTORE_PASSWORD` | Пароль keystore из шага 3 |
| `ANDROID_KEY_ALIAS` | `nova` |
| `ANDROID_KEY_PASSWORD` | Пароль key из шага 3 |

---

### Шаг 6. Запусти сборку AAB (2 минуты + 10 минут ожидания)

В репозитории GitHub:
1. Справа в боковой панели — раздел **Releases** → **Create a new release**
   (если нет — вкладка **Code** → справа **Releases** → **Create a new release**)
2. В поле **Choose a tag** напиши `v1.0.1`
3. Нажми **Create new tag: v1.0.1 on publish**
4. Заголовок: `Nova v1.0.1`
5. Описание: можешь оставить пустым (workflow сам сгенерирует)
6. Внизу: **Publish release**

GitHub запустит сборку. Можешь следить во вкладке **Actions**.

Через 7-10 минут на странице **Releases** появится готовый файл:
- `nova-v1.0.1.aab` ⭐ — для RuStore
- `nova-v1.0.1.apk` — для теста на телефоне

Скачай оба файла.

---

### Шаг 7. Загрузи AAB в RuStore (5 минут)

1. Открой https://console.rustore.ru/
2. Войди (нужна учётная запись разработчика, обычно регистрируется отдельно — там нужен ИП/ООО)
3. Найди свою заявку (которую отклонили) → **Загрузить новую версию**
4. Перетащи `nova-v1.0.1.aab`
5. Заполни поля (тексты ниже)
6. **Отправить на модерацию**

Срок: 1-3 рабочих дня.

## Если что-то не получается

**Сборка падает на GitHub Actions:**
- Открой Actions → красный run → раскрой упавший шаг → пришли мне ошибку

**Не могу найти где Secrets:**
- В репозитории (не в профиле!) → Settings → слева снизу: Secrets and variables → Actions

**RuStore не принимает AAB:**
- Возможно, тот же versionCode. В файле `android/app/build.gradle` найди `versionCode 1` и поменяй на `2`. Запушь, создай тег `v1.0.2`.

**Нет ИП/ООО для RuStore:**
- Без них RuStore мессенджеры не принимает. Открытие ИП через приложение «Налог ФЛ» — 0 руб, 3 рабочих дня.
