# Публикация Nova в RuStore — полная инструкция

Этот документ описывает весь процесс: от сборки AAB до прохождения модерации с первого раза.

## Что уже сделано в коде (проверка безопасности)

✅ **Cleartext HTTP запрещён** — `network_security_config.xml` разрешает только HTTPS
✅ **Бэкап данных запрещён** — `allowBackup=false` + `data_extraction_rules.xml`
✅ **Обфускация и минификация включены** — `minifyEnabled=true`, `shrinkResources=true`
✅ **Логи удалены из release** — ProGuard вырезает `Log.*` и `System.out.println`
✅ **WebView в защищённом режиме** — `webContentsDebuggingEnabled=false`, `allowMixedContent=false`
✅ **FileProvider экспорт закрыт** — `exported=false`
✅ **Экран согласия на обработку ПД** при первом запуске (152-ФЗ ст. 9)
✅ **Политика конфиденциальности внутри приложения** — Настройки → Политика конфиденциальности
✅ **Удаление аккаунта прямо из приложения** — Настройки → Удалить аккаунт (с двойным подтверждением)
✅ **Backend функция полного удаления данных** из всех 40+ таблиц БД

## Этап 1. Регистрация ОРИ в Роскомнадзоре (обязательно для мессенджеров)

С 2017 года все мессенджеры обязаны быть **Организаторами распространения информации**. Без записи в реестре ОРИ RuStore может отклонить заявку.

### Как зарегистрироваться

1. Зайди на сайт https://reestr.rkn.gov.ru/
2. Раздел «Подача уведомления о включении в реестр ОРИ»
3. Понадобится квалифицированная электронная подпись (КЭП) — оформляется в любом удостоверяющем центре (примерно 2-3 тыс. руб.)
4. Заполни форму: данные ИП/ООО, описание сервиса, URL приложения
5. Срок рассмотрения: 10 рабочих дней
6. Получаешь номер записи в реестре — он понадобится при заполнении карточки в RuStore

**Если ты физлицо без ИП** — это серьёзная проблема. Для мессенджера RuStore требует ИП или ООО. Открытие ИП через Госуслуги занимает 3 рабочих дня и стоит 800 руб госпошлины (или 0 руб при подаче через приложение «Налог ФЛ»).

## Этап 2. Сборка AAB (Android App Bundle) для RuStore

RuStore принимает **AAB** (предпочтительно) или APK. AAB — компактнее и Google Play такой же требует.

### Шаг 1. Подготовь GitHub репозиторий

См. `github-workflows/README.md` — настрой workflow один раз.

### Шаг 2. Сгенерируй ключ подписи (один раз)

На своём компьютере с установленным JDK 17 (https://adoptium.net/):

```bash
keytool -genkey -v -keystore nova-release.keystore -alias nova \
  -keyalg RSA -keysize 2048 -validity 10000
```

Ответь на вопросы. Пароли запиши надёжно — без них нельзя обновлять приложение.

### Шаг 3. Добавь Secrets в GitHub

GitHub → Settings → Secrets and variables → Actions → New repository secret:

| Имя | Значение |
|-----|----------|
| `ANDROID_KEYSTORE_BASE64` | `base64 nova-release.keystore \| tr -d '\n'` |
| `ANDROID_KEYSTORE_PASSWORD` | пароль от keystore |
| `ANDROID_KEY_ALIAS` | `nova` |
| `ANDROID_KEY_PASSWORD` | пароль от ключа |

### Шаг 4. Создай тег и запушь

```bash
git tag v1.0.0
git push origin v1.0.0
```

Через 7-10 минут на странице `github.com/твой-репо/releases` появятся:
- `nova-v1.0.0.aab` — для загрузки в RuStore ⭐
- `nova-v1.0.0.apk` — для теста на телефоне

## Этап 4. Заполнение карточки в RuStore Console

Зайди на https://console.rustore.ru/ → Опубликовать приложение → Загрузи AAB.

### Категория и теги

- **Категория:** Общение / Социальные сети
- **Возрастной рейтинг:** 12+ (для мессенджеров — стандартное значение)
- **Цена:** Бесплатно

### Описание (вставь как есть, отредактируй)

```
Nova — современный мессенджер для общения с близкими, друзьями и коллегами.

ВОЗМОЖНОСТИ:
• Личные и групповые чаты
• Голосовые сообщения и аудиозвонки
• Видеозвонки
• Каналы и сообщества
• Истории (24 часа)
• Стикеры и эмодзи
• Тёмная тема и кастомизация
• Push-уведомления
• Кошелёк для подарков и переводов между друзьями
• Поддержка нескольких аккаунтов

БЕЗОПАСНОСТЬ:
• Все данные хранятся на серверах в России
• Передача защищена TLS 1.2+
• Полное удаление аккаунта в один клик
• PIN-код и биометрия для блокировки приложения
• Соответствие 152-ФЗ «О персональных данных»

ВОЗРАСТНЫЕ ОГРАНИЧЕНИЯ: 12+
```

### Обоснование разрешений (раздел в карточке)

Здесь модератор RuStore смотрит ПОЧЕМУ ты просишь каждое опасное разрешение:

| Разрешение | Обоснование |
|------------|-------------|
| `CAMERA` | Съёмка фото и видео для отправки в чат |
| `RECORD_AUDIO` | Запись голосовых сообщений и аудиозвонки |
| `READ_MEDIA_IMAGES` | Прикрепление фото из галереи к сообщению |
| `READ_MEDIA_VIDEO` | Прикрепление видео из галереи к сообщению |
| `ACCESS_FINE_LOCATION` | Отправка геолокации в чат по запросу пользователя |
| `POST_NOTIFICATIONS` | Push-уведомления о новых сообщениях и звонках |
| `RECEIVE_BOOT_COMPLETED` | Восстановление подключения к push после перезагрузки |
| `FOREGROUND_SERVICE` | Активное аудио/видео-соединение во время звонка |
| `WAKE_LOCK` | Удержание соединения во время разговора |
| `VIBRATE` | Вибрация при входящих сообщениях и звонках |

⚠️ **Если какое-то разрешение ты не используешь** — лучше удалить из манифеста. Меньше разрешений = быстрее проходит модерация.

### Политика конфиденциальности (URL)

RuStore требует **публичный URL** на политику. Варианты:

1. **Лучше всего:** опубликовать политику на твоём сайте/лендинге
2. **Быстрый вариант:** GitHub Pages (бесплатно)
   - Создай файл `docs/privacy.html` с содержимым из `src/components/messenger/PrivacyPolicyPanel.tsx`
   - GitHub → Settings → Pages → Source: main branch /docs
   - URL станет: `https://username.github.io/repo/privacy.html`
3. **Платно (если ИП оформлен):** домен + хостинг 200 руб/месяц

### Контакты разработчика

- Email — обязательно, рабочий
- Сайт — желательно
- Телефон — необязательно

## Что делать если снова отказ

В письме модератора будет более конкретная причина. Частые сценарии:

### «Не работает регистрация»

Модератор тестирует через свои тестовые номера. Если ты используешь СМС-коды через платный шлюз, который требует деньги — модератор не сможет зарегистрироваться → отказ.

**Решение:** в комментарии к заявке укажи тестовый аккаунт:
```
Тестовый вход:
Телефон: +79991234567
Код подтверждения: 1234
```
В бэкенде сделай fallback: для конкретного номера всегда принимай код `1234` без отправки СМС.

### «Приложение содержит небезопасный код»

Модератор прогоняет AAB через сканер уязвимостей (обычно MobSF). Ищет:
- Хардкод секретов/токенов в коде
- Использование устаревших криптоалгоритмов (MD5, SHA1)
- Открытые компоненты (activity, service с `exported="true"` без причины)

**Решение:** проверь сам через https://mobsf.live/ — бесплатный онлайн-сканер.

### «Запрашиваются разрешения без обоснования»

Решение: убери все разрешения, которые не используются. Каждое оставшееся опиши в карточке.

### «Не работает удаление аккаунта»

Решение: запиши скринкаст процесса удаления и приложи к ответу модератору.

### «Нет согласия на обработку ПД»

Решение: запиши скринкаст экрана согласия при первом запуске и приложи к ответу.
